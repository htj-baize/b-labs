# kb/（知识库正文）

这里按主题分类存放可复用的工作流、规格、复盘与 runbook。

## 新增一篇文档怎么写

1) 复制模板：`kb/_TEMPLATE.md`
2) 填充以下最关键三段：
- **TL;DR**（30 秒读完）
- **解决方案**（可执行步骤 + 验证方法）
- **风险与注意事项**（避免踩坑/避免泄露/权限边界）

## 分类建议

- `git/`：代码仓、发布、CI、权限、密钥管理
- `neta/`：业务流程、规范、策略
- `recsys/`：推荐/分发/冷启等规格
- `schema/`：数据结构、映射、字段解释

> 约定：能公开的才入库；涉及 token、用户隐私、内部机密的内容先脱敏再写。
